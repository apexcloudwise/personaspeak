# Persona Schema (v1)

The `personas/*.yaml` files are the API between every consumer in this repo:
the desktop CLI, the Claude skill, and the Android app all read the same
files. Change this schema and you change all three — see AGENTS.md rule 2.

## Fields

| Field | Type | Required | Purpose |
|---|---|---|---|
| `schema_version` | int | no (assumed `1`) | Bump when the schema changes shape. |
| `name` | string | **yes** | Display name, e.g. `"Sir Humphrey Appleby"`. |
| `context` | string | no | Short parenthetical identifying who this is — becomes part of the prompt's first sentence, so include the leading space/paren: `" (the master of bureaucratic circumlocution from Yes Minister)"`. |
| `speech_patterns` | list of strings | **yes** | Stylistic traits: rhythm, tone, quirks. Aim for ~7. Each becomes a prompt bullet. |
| `vocabulary` | list of strings | no | Characteristic words/phrases. Aim for ~7. Joined into one prompt line. |
| `sample_lines` | list of strings | no | 3-4 example quotes. Tone/rhythm reference only — consumers must instruct the model not to copy them verbatim. |
| `notes` | string | no* | Caveats. *Required when `real_person: true`: declare the persona a stylistic homage to a public performance style, not a claim about the person's actual speech. |
| `real_person` | bool | no (default `false`) | Set `true` for personas based on real, living people. Makes `notes` mandatory (CI-enforced). |

## Prompt construction (normative)

All consumers must build the system prompt the same way, in this order:

1. Intro sentence: role statement + `name` + `context`.
2. `Voice characteristics:` header, then one `- ` bullet per `speech_patterns` entry.
3. If `vocabulary`: blank line, then a single line `Characteristic vocabulary/phrases to draw on: ` + comma-joined items.
4. If `sample_lines`: blank line, header line, then one quoted `- "…"` bullet per line.
5. If `notes`: blank line, then `Notes: ` + trimmed value.
6. Closing instruction: rewrite fully in-voice, preserve meaning, output only the rewritten text.

The reference implementation is `build_system_prompt()` in
`desktop/personaspeak.py`. The Kotlin implementation in
`android/core-personas` must produce **byte-identical output** for the same
YAML — that's what the golden tests in both languages pin
(fixtures in `tests/golden/`, generated by the Python reference).

## Content rules

- Personas are characters or performance-style homages. No personas designed
  to deceive (fake quotes from real people), defame, or harass.
- Keep sample lines original or short-quote-length; don't paste paragraphs of
  copyrighted script.

## Validation

`desktop/validate_personas.py` enforces the required fields, types, rough
list-length sanity, and `notes` whenever `real_person: true`. CI runs it on
every push/PR.
